## Deployment source of truth

The EC2 host now runs from a GitHub clone. Deployment updates should come from `origin/main`, not from manual file transfer.

Current flow:

```bash
git push origin main
ssh -i "quant-exchange.pem" ec2-user@16.59.150.9 '
  cd ~/exchange-v2 &&
  git pull --ff-only &&
  cd exchange &&
  source "$HOME/.cargo/env" &&
  cargo build --release &&
  sudo systemctl restart exchange &&
  sudo systemctl status exchange --no-pager
'
```
