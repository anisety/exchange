## Current state

- The exchange backend is live on EC2.
- The Next.js client is deployed on ECS/Fargate behind an ALB.
- The ALB hostname is `exchange-client-alb-1466111370.us-east-2.elb.amazonaws.com`.
- The public hostname is `https://exchange.jamesxu.dev`.
- HTTPS is terminated at the ALB with ACM.

## Environment variables

Use these for a deployed client:

```text
NEXT_PUBLIC_EXCHANGE_HTTP_URL=https://exchange.jamesxu.dev
NEXT_PUBLIC_EXCHANGE_WS_URL=wss://exchange.jamesxu.dev/ws
NEXT_PUBLIC_EXCHANGE_MARKETS=BTC-USD,ETH-USD,SOL-USD
```

`NEXT_PUBLIC_EXCHANGE_API_KEY` is optional and mainly useful for local testing or a pinned kiosk-like client. Admin access is now determined by backend validation, not a client-side key list.
