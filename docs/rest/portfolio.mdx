---
title: "Get Portfolio"
api: "GET /api/v1/portfolio"
description: "Read your current position limit and all current positions in one response."
---

Call this endpoint to bootstrap your account panel and confirm your current limit before placing new trades.

<RequestExample>

```bash Request
curl https://exchange.jamesxu.dev/api/v1/portfolio \
  -H "x-api-key: YOUR_API_KEY"
```

</RequestExample>

<ResponseExample>

```json 200 Response
{
  "trader_id": "c7f8c572-8a50-4dbd-a1e1-9f452774b636",
  "position_limit": 1000,
  "positions": [
    {
      "market": "BTC-USD",
      "net_quantity": 40,
      "average_entry_price": 1,
      "realized_pnl": 0,
      "updated_at": "2026-03-19T21:21:34.741319Z"
    }
  ]
}
```

</ResponseExample>

## Headers

<ParamField header="x-api-key" type="string" required>
  Assigned competitor API key.
</ParamField>

## Response fields

<ResponseField name="trader_id" type="string" required>
  Stable identity of the authenticated trader.
</ResponseField>

<ResponseField name="position_limit" type="i64 | null" required>
  Fixed per-market worst-case net limit for standard competitors. Privileged admin traders return `null`.
</ResponseField>

<ResponseField name="positions" type="object[]" required>
  Current position set for the trader.
</ResponseField>

<ResponseField name="positions[].market" type="string" required>
  Market identifier for the position.
</ResponseField>

<ResponseField name="positions[].net_quantity" type="i64" required>
  Signed position size.
</ResponseField>

<ResponseField name="positions[].average_entry_price" type="u64 | null" required>
  Weighted average entry price for the open quantity.
</ResponseField>

<ResponseField name="positions[].realized_pnl" type="i64" required>
  Realized PnL already locked in from closed or settled quantity.
</ResponseField>

## Notes

- Use this endpoint when you want a single account bootstrap call instead of separate user and position reads.
- Unrealized PnL is not returned here.
- This is the recommended account bootstrap call after login, reconnect, or settlement.
