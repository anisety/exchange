---
title: "Get Open Orders"
api: "GET /api/v1/open-orders"
description: "Read your currently resting orders, optionally filtered by market."
---

Use this endpoint to rebuild visible working orders after login, reconnect, or resync.

<RequestExample>

```bash Request
curl 'https://exchange.jamesxu.dev/api/v1/open-orders?market=BTC-USD' \
  -H "x-api-key: YOUR_API_KEY"
```

</RequestExample>

<ResponseExample>

```json 200 Response
[
  {
    "id": "66377526-7b98-485a-8c40-7024e68fa3c5",
    "trader_id": "9d64e278-8b56-46e9-9cab-18c5b22f2fb9",
    "market": "BTC-USD",
    "side": "BUY",
    "price": 100,
    "quantity": 2,
    "remaining": 1,
    "created_at": "2026-03-18T15:28:01.342510Z"
  }
]
```

</ResponseExample>

## Headers

<ParamField header="x-api-key" type="string" required>
  Assigned competitor API key.
</ParamField>

## Query parameters

<ParamField query="market" type="string">
  Optional market filter, for example `BTC-USD`.
</ParamField>

## Response fields

<ResponseField name="[]" type="object[]" required>
  Currently resting open orders owned by the authenticated trader.
</ResponseField>

<ResponseField name="[].id" type="string" required>
  Stable order identifier used for amend and cancel requests.
</ResponseField>

<ResponseField name="[].market" type="string" required>
  Market identifier.
</ResponseField>

<ResponseField name="[].side" type="BUY | SELL" required>
  Order side.
</ResponseField>

<ResponseField name="[].price" type="u64" required>
  Limit price.
</ResponseField>

<ResponseField name="[].quantity" type="u64" required>
  Original order quantity.
</ResponseField>

<ResponseField name="[].remaining" type="u64" required>
  Quantity still resting on the book.
</ResponseField>

## Notes

- Only currently open resting orders are returned.
- This endpoint is the recommended source of truth after a reconnect.
- A fully filled or canceled order disappears from this list.
