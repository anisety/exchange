---
title: "Cancel Order"
description: "Cancel an existing open order over WebSocket."
---
Send this message when you want to remove an open order from the book.

<RequestExample>

```json Client Message
{
  "op": "cancel_order",
  "request_id": "req-2",
  "order_id": "66377526-7b98-485a-8c40-7024e68fa3c5"
}
```

</RequestExample>

<ResponseExample>

```json Ack Response
{"type":"ack","op":"cancel_order","request_id":"req-2"}
```

</ResponseExample>

## Client fields

- `op` must be `cancel_order`.
- `request_id` is client-defined and echoed back in `ack` or `reject`.
- `order_id` is the target open order.

## Failure behavior

Failures are returned as `reject` messages. Common reasons are:

- `unauthenticated`
- `order_not_found`
- `order_not_owned`

## Notes

- The server emits an `order_state` message if the cancel succeeds.
