---
title: "Unsubscribe Data Stream"
description: "Stop receiving the aggregated public market-data stream on the current socket."
---

Use `unsubscribe` when you want to stop receiving public market data for the current subscription.

<RequestExample>

```json Client Message
{"op":"unsubscribe","channel":"data","market":"BTC-USD"}
```

</RequestExample>

## Client fields

- `op` must be `unsubscribe`.
- `channel` must be `data`.
- `market` is the subscribed market you want to stop receiving.

## Notes

- Unsubscribe affects only the specified market-data subscription.
- Authentication state on the socket is unchanged.
